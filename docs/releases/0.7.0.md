# v0.7.0 - stronger open-source-informed integrity signals

This backward-compatible feature release strengthens the `os_integrity` probe with additional
raw root, hook, and emulator observations, informed by widely used open-source device-integrity
detectors. The SDK continues to return raw evidence only: it does not calculate a score, emit a
trusted/untrusted verdict, contact Google Play, or upload data.

## Highlights

### Stack-trace hook probe (Android)

`os_integrity` adds a third, independent hook-detection method alongside the loaded-class lookup
and the `/proc/self/maps` scan:

- `hookStackFrameFound` — a captured stack trace contains Xposed, LSPosed, EdXposed, or Substrate
  bridge frames, or `com.android.internal.os.ZygoteInit` appears more than once (a re-injected
  Zygote tell);
- `hookStackFrames` — the raw matched `class.method` frames, for backend triage.

The probe throws and immediately catches an exception to read the current call stack; it performs
no reflection into third-party classes and initializes nothing.

### Broader Android root evidence

- `suOnPath` — `which su` resolved an `su` binary on the process `PATH`, orthogonal to the
  file-existence scan;
- the `su`-binary scan now also covers every directory in the process `PATH` plus
  `/system_ext/bin` and `/cache`;
- additional known root/injection artifact paths (`/data/adb/magisk.db`, `/sbin/.magisk`,
  `/cache/.disable_magisk`, `/system/framework/XposedBridge.jar`, `/data/local/tmp/re.frida.server`,
  `/system/app/SuperSU.apk`, `/system/app/Kinguser.apk`);
- `dangerousAppFound` — a patcher / ROM-manager / in-app-purchase-spoofer package is installed
  (e.g. Lucky Patcher, Freedom);
- `rootCloakingAppFound` — a root-cloaking / root-hiding package is installed (e.g. RootCloak,
  Hide My Root).

The two new app categories are queried only through the finite `<queries>` allow-list. The library
still requests no `QUERY_ALL_PACKAGES` permission and adds none to the host manifest.

### Wider emulator coverage

- the emulator classifier now recognizes the **MuMu** emulator family and reads `Build.HOST`,
  emitting a `host:qemu` build marker and a `buildbot_host` device-farm marker.

### iOS injection and symlink coverage

The iOS dyld-image scan and jailbreak symlink probe recognize more artifacts
(`SSLKillSwitch2`, `0Shadow`, `FlyJB`, `Cephei`, `Electra`, `AppSyncUnified`, `WeeLoader`,
`libsparkapplist`, and additional `/var`/`/usr` symlink paths). These extend the existing
`injectedLibraryNames` and `symbolicLinksSuspicious` observations; no new field is introduced on iOS.

## Backend contract guidance

- Treat every new field as optional and attacker-influenced; preserve missing versus `false`.
- `hookStackFrames`, `dangerousAppFound`, and `rootCloakingAppFound` are Android-only and omitted on
  iOS.
- Use these as explainable context fused alongside authenticated server data and platform
  attestation, never as a single-signal block.
- No existing field is removed or renamed.

## Verification

- Root Jest suite: 105 tests passed (including the drift guard between `KnownAppLists` and the
  manifest `<queries>` for the new package categories).
- TypeScript, TurboModule contract, package, and GitHub Pages verification passed.
- Pure-Kotlin unit tests cover the new stack-frame classifier.

The implementation adds no npm runtime, Gradle runtime, CocoaPods, or bundled binary dependencies;
no JNI/NDK, network requests, persistent identifiers, `QUERY_ALL_PACKAGES`, runtime permission
prompts, or Apple Required-Reason API declarations were added.

## Upgrade

```sh
npm install react-native-device-risk-signals@0.7.0
```

For iOS applications, refresh CocoaPods after updating:

```sh
cd ios && bundle exec pod install
```

No existing field is removed or renamed; existing integrations retain their previous behavior.
