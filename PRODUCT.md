# Product

## Register

product

## Platform

adaptive

## Users

React Native developers evaluating the library before integrating it into an application. They use
the demo on a real device or emulator to understand which signals are available and how the returned
payload is structured.

## Product Purpose

Provide a runnable reference application that collects device risk signals on demand and presents the
complete raw result in a form that is easy to inspect. Success means a developer can clone the
repository, launch the demo, run collection, and understand the output without reading native source.

## Positioning

The transparent, device-local way to inspect exactly what the library observes.

## Brand Personality

A focused security and developer tool: precise, technical, and trustworthy.

## Anti-references

Do not imitate a hacker terminal. Avoid neon-on-black styling, cyberpunk decoration, fake command
prompts, and theatrical security language.

## Design Principles

- Put the collected evidence first; decoration must never compete with the payload.
- Make collection state and platform availability unambiguous.
- Explain sensitive behavior in plain language before the developer triggers it.
- Keep the interface useful on both compact phones and larger development devices.

## Accessibility & Inclusion

The demo intentionally supports a light appearance only.
